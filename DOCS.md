## Требования

- **Node.js** 18 или выше (рекомендуется 20+)
- **npm** 9 или выше

Проверить установленные версии:

```bash
node -v
npm -v
```

Если Node.js не установлен, скачайте с https://nodejs.org (вариант LTS).

## Установка

Откройте терминал, перейдите в папку проекта и установите зависимости:

```bash
cd pesnohorky-latest
npm install
```

Одна команда устанавливает всё: React, Vite (фронтенд), Express, SQLite, Multer (бэкенд).

---

## Запуск

Для работы нужны **два процесса** — бэкенд и фронтенд. Откройте два окна терминала.

### Терминал 1 — бэкенд (API-сервер + админка)

```bash
npm start
```

Сервер стартует на **http://localhost:3001**. В консоли появится:

```
Сервер запущен: http://localhost:3001
Админка педагогов: http://localhost:3001/admin/teachers.html
Админка новостей:  http://localhost:3001/admin/news.html
```

При первом запуске автоматически создаётся файл базы данных `backend/database.sqlite` и папка `backend/uploads/` для загруженных изображений.

### Терминал 2 — фронтенд (React-приложение)

```bash
npm run dev
```

Фронтенд стартует на **http://localhost:5173**. Все запросы к `/api/*` автоматически проксируются на бэкенд (порт 3001) через настройку в `vite.config.ts`.

> **Важно:** бэкенд должен быть запущен до или одновременно с фронтендом, иначе страницы педагогов и новостей будут показывать ошибку загрузки.

### Быстрый старт (обе команды)

Если у вас терминал с поддержкой фоновых процессов:

```bash
npm start &
npm run dev
```

## Админка

Админка — это отдельные HTML-страницы, доступные через бэкенд-сервер. Они не зависят от React-фронтенда.

### Педагоги

**URL:** http://localhost:3001/admin/teachers.html

На странице:

1. **Форма создания педагога** с полями:
   - **Имя** (обязательно) — ФИО полностью
   - **Биография** (обязательно) — текстовый редактор с форматированием (Quill)
   - **Фото** — загрузка файла (drag & drop или клик), изображение сохраняется на сервер

2. **Таблица педагогов** — все добавленные педагоги с миниатюрой фото, датой создания и кнопкой удаления

### Новости

**URL:** http://localhost:3001/admin/news.html

На странице:

1. **Форма создания новости** с полями:
   - **Заголовок** (обязательно) — краткое название
   - **Содержание** (обязательно) — текстовый редактор с форматированием (Quill)
   - **Дата** (обязательно) — выбор из календаря
   - **Изображение** — загрузка файла

2. **Таблица новостей** — все добавленные новости с миниатюрой, датой, превью текста и кнопкой удаления

Навигация между разделами — ссылки вверху каждой страницы.

### Загрузка изображений

Изображения загружаются через drag & drop область или по клику. Процесс:

1. Выбираете или перетаскиваете файл
2. Файл загружается на сервер через `POST /api/upload`
3. Появляется превью загруженного изображения
4. Область подсвечивается зелёной рамкой
5. При отправке формы ссылка на загруженный файл сохраняется в БД

Ограничения:
- Максимальный размер: **10 МБ**
- Форматы: JPG, JPEG, PNG, GIF, WebP, SVG
- Файлы сохраняются в `backend/uploads/` с уникальными именами

### Текстовый редактор

Обе формы используют **Quill** — визуальный редактор с панелью инструментов:

**В форме педагога:**
- Жирный, курсив, подчёркивание
- Нумерованный и маркированный списки
- Ссылки

**В форме новости (расширенный):**
- Заголовки (H2, H3)
- Жирный, курсив, подчёркивание, зачёркнутый
- Нумерованный и маркированный списки
- Цитаты, ссылки

Контент сохраняется в БД как **HTML-разметка** и рендерится на фронтенде через `dangerouslySetInnerHTML`.

## API-справочник

Базовый URL: `http://localhost:3001`

### API: Педагоги

#### Список педагогов

```
GET /api/teachers
```

Ответ:
```json
[
  {
    "id": 1,
    "name": "Бурдина Алена Максимовна",
    "photo_url": "/uploads/a1b2c3d4e5f6.jpg"
  },
  {
    "id": 2,
    "name": "Иванова Анна Владимировна",
    "photo_url": null
  }
]
```

#### Детально о педагоге

```
GET /api/teachers/:id
```

Ответ:
```json
{
  "id": 1,
  "name": "Бурдина Алена Максимовна",
  "bio": "<p>Педагог дополнительного образования, руководитель хореографической студии <strong>«Орбита»</strong></p>",
  "photo_url": "/uploads/a1b2c3d4e5f6.jpg",
  "created_at": "2026-04-05 10:30:00"
}
```

Поле `bio` содержит HTML-разметку (из Quill-редактора).

#### Создание педагога

```
POST /api/admin/teachers
Content-Type: application/json

{
  "name": "Бурдина Алена Максимовна",
  "bio": "<p>Педагог дополнительного образования...</p>",
  "photo_url": "/uploads/a1b2c3d4e5f6.jpg"
}
```

- `name` — обязательно
- `bio` — обязательно (HTML-строка)
- `photo_url` — необязательно

Ответ: `{ "id": 1 }` (статус 201)

#### Удаление педагога

```
DELETE /api/admin/teachers/:id
```

Ответ: `{ "ok": true }`

### API: Новости

#### Список новостей

```
GET /api/news
```

Ответ:
```json
[
  {
    "id": 1,
    "title": "Весенний калейдоскоп",
    "date": "2026-03-15",
    "image_url": "/uploads/f6e5d4c3b2a1.jpg",
    "preview": "<p>15 марта в ДК «Родина» прошел ежегодный фестиваль детского творчества...</p>"
  }
]
```

Поле `preview` — первые 200 символов из `content` (может содержать незакрытые HTML-теги).

#### Детально о новости

```
GET /api/news/:id
```

Ответ:
```json
{
  "id": 1,
  "title": "Весенний калейдоскоп",
  "content": "<p>15 марта в ДК «Родина» прошел ежегодный фестиваль...</p><h2>Программа</h2><ul><li>Вокал</li><li>Хореография</li></ul>",
  "date": "2026-03-15",
  "image_url": "/uploads/f6e5d4c3b2a1.jpg"
}
```

Поле `content` содержит полную HTML-разметку.

#### Создание новости

```
POST /api/admin/news
Content-Type: application/json

{
  "title": "Весенний калейдоскоп",
  "content": "<p>15 марта в ДК «Родина» прошел...</p>",
  "date": "2026-03-15",
  "image_url": "/uploads/f6e5d4c3b2a1.jpg"
}
```

- `title` — обязательно
- `content` — обязательно (HTML-строка)
- `date` — обязательно (формат `YYYY-MM-DD`)
- `image_url` — необязательно

Ответ: `{ "id": 1 }` (статус 201)

#### Удаление новости

```
DELETE /api/admin/news/:id
```

Ответ: `{ "ok": true }`

### API: Загрузка файлов

```
POST /api/upload
Content-Type: multipart/form-data

Поле: file (файл изображения)
```

Ответ:
```json
{
  "url": "/uploads/a1b2c3d4e5f6a7b8c9d0e1f2.jpg"
}
```

Загруженный файл доступен по адресу `http://localhost:3001/uploads/имя_файла`.

Ограничения:
- Максимальный размер файла: 10 МБ
- Форматы: `.jpg`, `.jpeg`, `.png`, `.gif`, `.webp`, `.svg`

## Пошаговые инструкции

### Как добавить педагога

1. Убедитесь, что бэкенд запущен (`npm start`)
2. Откройте **http://localhost:3001/admin/teachers.html**
3. Заполните форму:
   - **Имя** — введите полное ФИО, например: `Бурдина Алена Максимовна`
   - **Биография** — опишите педагога в текстовом редакторе. Используйте форматирование: жирный для должностей, списки для достижений. Пример:
     - Должность: **Педагог дополнительного образования**
     - Руководитель хореографической студии «Орбита»
     - Первая квалификационная категория
   - **Фото** — перетащите фотографию педагога в область загрузки (или кликните на неё). Дождитесь появления превью
4. Нажмите **Создать**
5. Появится зелёное уведомление «Педагог добавлен!», форма очистится
6. Педагог сразу появится:
   - В таблице ниже (на странице админки)
   - На фронтенде: **http://localhost:5173/about-info/teachers-staff** (карточка в сетке)
   - По прямой ссылке: **http://localhost:5173/teachers/{id}** (детальная страница)

### Как добавить новость

1. Убедитесь, что бэкенд запущен (`npm start`)
2. Откройте **http://localhost:3001/admin/news.html**
3. Заполните форму:
   - **Заголовок** — краткое название, например: `Весенний калейдоскоп`
   - **Содержание** — полный текст новости в редакторе. Можно использовать заголовки, списки, цитаты, ссылки. Разбивайте текст на абзацы
   - **Дата** — выберите дату из календаря
   - **Изображение** — перетащите главное изображение в область загрузки. Дождитесь превью
4. Нажмите **Создать**
5. Появится зелёное уведомление «Новость добавлена!»
6. Новость сразу появится:
   - В таблице ниже (на странице админки)
   - На фронтенде: **http://localhost:5173/news** (в ленте новостей)
   - По прямой ссылке: **http://localhost:5173/news/{id}** (полная новость)

### Как удалить запись

1. На странице админки найдите запись в таблице
2. Нажмите красную кнопку **Удалить**
3. Подтвердите удаление в всплывающем окне
4. Запись исчезнет из таблицы и с фронтенда

> Внимание: удаление необратимо. Загруженные файлы изображений остаются на диске, но запись из БД удаляется.

## Фронтенд-интеграция

React-страницы получают данные из API через функции в `src/lib/api.ts`:

| Страница | URL | Что делает |
|----------|-----|------------|
| **TeachersStaffPage** | `/about-info/teachers-staff` | Вызывает `GET /api/teachers`, показывает карточки педагогов в CSS Grid. Клик ведёт на `/teachers/:id`. PDF-ссылки на документы сохранены |
| **TeacherDetailPage** | `/teachers/:id` | Берёт `id` из URL через `useParams()`, вызывает `GET /api/teachers/:id`. Показывает фото, имя, биографию (HTML рендерится через `dangerouslySetInnerHTML`) |
| **NewsLandingPage** | `/news` | Вызывает `GET /api/news`, распределяет по секциям: первые 3 — средние карточки, 4-я — главная, 5–6 — компактные, остальные — события. Если API недоступен — показывает заглушки с дефолтными данными |
| **NewsDetailPage** | `/news/:slug` | Берёт `id` из URL, вызывает `GET /api/news/:id`. Показывает заголовок, дату, изображение, контент (HTML через `dangerouslySetInnerHTML`) |

Прокси настроен в `vite.config.ts`:

```ts
server: {
  proxy: {
    '/api': 'http://localhost:3001',
  },
},
```

В dev-режиме фронтенд (порт 5173) прозрачно проксирует все `/api/*` запросы на бэкенд (порт 3001). Никаких абсолютных URL в React-коде нет — все запросы идут на `/api/...`.

## База данных

Используется **SQLite** — файловая БД, не требует отдельного сервера.

**Файл:** `backend/database.sqlite` (создаётся автоматически при первом запуске, добавлен в `.gitignore`)

### Таблицы

**teachers:**

| Столбец    | Тип     | Описание                              |
|------------|---------|---------------------------------------|
| id         | INTEGER | Первичный ключ, автоинкремент         |
| name       | TEXT    | ФИО педагога (обязательно)            |
| bio        | TEXT    | Биография в формате HTML (обязательно)|
| photo_url  | TEXT    | Путь к фото, например `/uploads/...` (может быть NULL) |
| created_at | TEXT    | Дата создания (заполняется автоматически) |

**news:**

| Столбец   | Тип     | Описание                              |
|-----------|---------|---------------------------------------|
| id        | INTEGER | Первичный ключ, автоинкремент         |
| title     | TEXT    | Заголовок (обязательно)               |
| content   | TEXT    | Полный текст в формате HTML (обязательно) |
| date      | TEXT    | Дата публикации, формат YYYY-MM-DD (обязательно) |
| image_url | TEXT    | Путь к изображению (может быть NULL)  |

### Сброс базы данных

Чтобы начать с чистой БД, остановите бэкенд и удалите файл:

```bash
# Остановите бэкенд (Ctrl+C), затем:
rm backend/database.sqlite
npm start
```

Таблицы пересоздадутся автоматически (пустые).

### Просмотр содержимого БД

Если нужно посмотреть данные напрямую, можно использовать sqlite3:

```bash
sqlite3 backend/database.sqlite
.tables
SELECT * FROM teachers;
SELECT * FROM news;
.quit
```

## Деплой на сервер (продакшен)

### 1. Сборка фронтенда

```bash
npm run build
```

Результат — папка `dist/` с готовыми статическими файлами.

### 2. Что копировать на сервер

```
backend/          — весь бэкенд (index.js, db.js, routes/, uploads/)
dist/             — собранный фронтенд
public/admin/     — админка
package.json      — зависимости
package-lock.json — точные версии
```

### 3. Установка на сервере

```bash
npm install --production
npm start
```

### 4. Настройка nginx

Пример конфигурации nginx для проксирования:

```nginx
server {
    listen 80;
    server_name pesnohorki.example.com;

    # React SPA
    root /path/to/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # API и загруженные файлы
    location /api/ {
        proxy_pass http://127.0.0.1:3001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /uploads/ {
        proxy_pass http://127.0.0.1:3001;
    }

    # Админка
    location /admin/ {
        proxy_pass http://127.0.0.1:3001;
    }
}
```

### 5. Автозапуск (PM2)

Чтобы бэкенд автоматически перезапускался:

```bash
npm install -g pm2
pm2 start backend/index.js --name pesnohorky
pm2 save
pm2 startup
```

## FAQ и решение проблем

### Страницы педагогов/новостей пустые или показывают «Загрузка...»

Бэкенд не запущен. Откройте отдельный терминал и выполните `npm start`. Проверьте:

```bash
curl http://localhost:3001/api/teachers
```

Должен вернуть JSON (массив, возможно пустой `[]`).

### Ошибка при загрузке изображения

- Убедитесь, что файл не превышает 10 МБ
- Убедитесь, что формат — JPG, PNG, GIF, WebP или SVG
- Проверьте, что папка `backend/uploads/` существует и доступна для записи

### Ошибка «Cannot find module sqlite3» или «Cannot find module multer»

Зависимости не установлены. Выполните:

```bash
npm install
```

На Windows, если sqlite3 не компилируется:

```bash
npm install --global windows-build-tools
npm install
```

### Порт 3001 занят

Измените порт в `backend/index.js` (строка `const PORT = 3001`) и обновите прокси в `vite.config.ts`:

```ts
proxy: {
  '/api': 'http://localhost:НОВЫЙ_ПОРТ',
},
```

### Данные пропали после перезапуска

Данные хранятся в `backend/database.sqlite`. Если файл на месте — данные сохранены. Если файл был удалён — данные утеряны. Рекомендуется периодически копировать этот файл как бэкап.

### Загруженные фото не отображаются на фронтенде

В dev-режиме фото доступны через бэкенд (`http://localhost:3001/uploads/...`). Прокси Vite не перенаправляет `/uploads/` автоматически. Два варианта решения:

1. Добавить прокси в `vite.config.ts`:
   ```ts
   proxy: {
     '/api': 'http://localhost:3001',
     '/uploads': 'http://localhost:3001',
   },
   ```

2. В продакшене nginx проксирует `/uploads/` напрямую (см. раздел «Деплой»).

### Форматирование текста не отображается на сайте

Биографии и новости сохраняются как HTML. На фронтенде они рендерятся через `dangerouslySetInnerHTML`. Если вместо форматированного текста видны теги — проверьте, что React-компонент использует `dangerouslySetInnerHTML`, а не обычный текстовый вывод.
